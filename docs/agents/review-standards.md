# Review and Output Standards

Standards for reviews, discussions, and output formatting.

## Comparative Analysis

When proposing multiple options, compare:

- implementation cost
- operational complexity
- maintainability
- scalability
- migration risk
- security implications

## Decision Trace

For significant decisions, briefly document:

- chosen approach
- rejected alternatives
- reasoning

Avoid excessive documentation overhead.

## Structure and Readability

Use appropriate headings, lists, code blocks, and tables when useful.

Preserve readability and navigability in generated outputs.

## Technical Clarity

- Avoid unnecessary ambiguity
- Use conditional language only when uncertainty genuinely exists
- Keep explanations concise but sufficiently actionable

## Response Density

- Keep simple-task responses concise
- Use structured detail for complex tasks
- Avoid unnecessary verbosity
